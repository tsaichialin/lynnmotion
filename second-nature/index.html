SECOND NATURE

Concept, Content, Code: Building a Fashion Campaign Page with AI

Part 1

Concept: Ideation and Web Architecture with AI

The concept centers on a fictional fashion brand called SECOND NATURE. The campaign page follows a protagonist through a series of events, using 16 distinct scenes to showcase clothing through cinematic, narrative-driven fashion photography.

The main characters — a woman and a dark creature — are carried over from a previous project. Drawing inspiration from film, the woman explores a series of suspenseful, cinematic scenarios dressed in SECOND NATURE pieces.

The process started with conversations in Perplexity, using both Claude and ChatGPT to work through the content, structure, and execution details. After several rounds of discussion, the direction gradually came into focus.

Fig. 1. Conversations with LLMs — from overall architecture down to the details

Beyond content, the technical direction was also established at this stage. Early testing was done to verify whether the concept was actually buildable.

Various interaction techniques were prototyped using Claude's web interface. Any promising visual or motion effects were exported as HTML and later imported into Claude Code for refinement.

Several key decisions were locked in during this phase: a single-page layout, a 5-second hero video at the top, and a scroll-driven video playback — where the video timeline advances as the user scrolls.

To match the detective story tone, Claude was asked to suggest suitable interaction techniques — including transitions between scenes. Two features were selected:

  • Redacted Reveal — a text effect that simulates classified information being uncovered. Each line of text is covered by a black overlay that slides away in sequence after a scene transition, like a declassified document being read for the first time.
  • Hotspot Annotation — each scene features one clickable marker on the outfit that expands and collapses to reveal details about the garment.
Fig. 2. The Redacted Reveal animation in action

After multiple rounds of back-and-forth with Claude — refining direction, testing motion ideas, and finalizing interaction logic — the content and approach were solid enough to move into visual production.

Part 2

Content: Images and Video with AI

Two image models were tested to explore visual directions. Flux leaned toward a graphic, fashion-illustration aesthetic (Fig. 3), while Nano Banana produced more realistic, cinematic results (Fig. 4) — a better fit for this project.

Fig. 3. Flux2 Pro — graphic and design-forward · Fig. 4. Nano Banana Pro/2 — cinematic and true-to-life

Nanona was chosen to generate all images. Each image went through at least three rounds of iteration — a time-consuming but worthwhile process.

Step 1

Finding Scenes and Story Direction

The process started with a 3×3 grid — generating 9 images at once to quickly explore scene possibilities and save time. At this stage, the main criteria were relevance to the theme and whether each image felt like it had a story behind it.

The following prompt was used to place the character across a variety of settings:

Create a full Instagram-style composed of nine different portrait images. Each of the 9 images should present a unique concept, outfit, pose, and environment that fits a stylish, modern aesthetic.

Include a mix of:

  • Cinematic portraits
  • Fashion/streetwear scenes
  • Close-up beauty shots
  • Minimalist studio portraits
Fig. 5. Generating a 3×3 grid of 9 images in one go using Nano Banana Pro/2 on Freepik
Step 2

Multiple Shots Within the Same Scene

After generating a large pool of scenes, the best images were selected and expanded into a 2×3 grid of 6 shots. The focus here shifted from scene variety to variations within the same scene — different angles, emotional beats, and story moments. Note: the phrase "20-second cinematic clip" in the prompt below isn't asking for a video — it's instructing Nano Banana to think in terms of cinematic narrative logic across a series of still images.

Prompt: "Expand the image into a 20-second cinematic clip with a clear theme and emotional progression (setup → build → turn → payoff)."

Fig. 6. Generating a 2×3 grid of 6 shots using Nano Banana Pro/2 on Freepik
Step 3

Upscaling and Refining

Promising images from the previous step were upscaled, then refined across multiple variations. The strongest version of each was selected for the final site.

Fig. 7. Selected images go through continuous rounds of refinement

With all 16 images finalized, attention turned to the hero video at the top of the page.

The video serves as the background for the Hero Section — the large banner at the top of the page, the first thing visitors see. It shows the woman receiving an invitation letter, echoing the concept of being drawn into each event.

Fig. 8. Generating the hero video using Seedance2 on the Dreamina platform

With copy, images, and video all in place, it was time to start building with Claude Code.

Part 3

Code: Building and Publishing with Claude Code

The initial plan was to use Google Stitch for visual design and Claude Code for the code. But Stitch felt too limiting for precise detail work, so the approach was simplified: Claude Code combined with web-specific Skills.

After installing Claude Code locally, a quick test was run through the CLI — a simple prompt to generate a basic webpage. It worked as expected.

For the actual build, Claude Code was moved into an IDE where the graphical interface is more manageable. Both Antigravity and VS Code were considered — Antigravity is a VS Code fork with a very similar interface — but VS Code won out for familiarity.

With the Claude Code extension installed in VS Code and a set of design Skills loaded as reference, the build could begin. A Skill here is a predefined visual specification document — it gives Claude a design system to follow when generating the page.

Having a Skill doesn't guarantee a good-looking result. Aesthetic judgment and UI/UX experience are still essential for getting the details right. AI lowers the barrier to entry, but the quality of the outcome still depends on the person making the calls.

Fig. 9. Claude Code running in VS Code. The diff view shows before (left) and after (right) — making every change easy to review. Designers without a coding background may not recognize CSS or JS, but that's perfectly fine. Claude Code handles all of it automatically.

Around the 80% mark, describing fine-tuned adjustments through prompts started to feel inefficient. Using Figma MCP was briefly considered for detail work, but the UI was simple enough that no formal design handoff was needed. Direct edits to CSS and JS turned out to be faster than prompting Claude Code for each small change.

Claude Code is powerful, but Anthropic places usage limits on it. The 5-hour session limit and 7-day usage cap came up repeatedly throughout the build, requiring frequent pauses while waiting for the session counter to reset.

The biggest difference AI makes in web production is speed of problem-solving. What used to mean long detours through Google Search and Stack Overflow can now be handled in a conversation with an LLM — leaving more time and energy for experimentation and refinement. The finished page was published to GitHub Pages to make it publicly accessible.

That covers the full process — from the initial idea to a live, published page, built in collaboration with AI.

Despite the heavy use of AI tools throughout, the human element remains irreplaceable. Every meaningful decision — the theme, the visual direction, the interaction design — required human judgment. As Agentic Workflows mature, it may eventually be possible to automate the entire pipeline. But AI tends toward the aesthetic average, producing results that can feel visually generic. For a project like this — unconventional and narrative-driven — human intervention isn't optional, it's what makes the difference.
