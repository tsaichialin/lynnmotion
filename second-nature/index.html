Concept, Content, Code: Building a Fashion Campaign Page with AI

Part 1

Concept: Ideation and Web Architecture with AI

The concept centers on a fictional fashion brand called SECOND NATURE. The campaign page follows a protagonist through a series of events, using 16 distinct scenes to showcase clothing through cinematic, narrative-driven fashion photography.

The main characters — a woman and a dark creature — are carried over from a previous project. Drawing inspiration from film, the woman explores a series of suspenseful, cinematic scenarios dressed in SECOND NATURE pieces.

The process started with conversations in Perplexity, using both Claude and ChatGPT to work through the content, structure, and execution details. After several rounds of discussion, the direction gradually came into focus.

Conversations with LLMs — from overall architecture down to the details

Beyond content, the technical direction was also established at this stage. Early testing was done to verify whether the concept was actually buildable.

Various interaction techniques were prototyped using Claude's web interface. Any promising visual or motion effects were exported as HTML and later imported into Claude Code for refinement.

Several key decisions were locked in during this phase: a single-page layout, a 5-second hero video at the top, and a scroll-driven video playback — where the video timeline advances as the user scrolls.

To match the detective story tone, Claude was asked to suggest suitable interaction techniques — including transitions between scenes. Two features were selected:

  • Redacted Reveal — a text effect that simulates classified information being uncovered. Each line of text is covered by a black overlay that slides away in sequence after a scene transition, like a declassified document being read for the first time.
  • Hotspot Annotation — each scene features one clickable marker on the outfit that expands and collapses to reveal details about the garment.
The Redacted Reveal animation in action

After multiple rounds of back-and-forth with Claude — refining direction, testing motion ideas, and finalizing interaction logic — the content and approach were solid enough to move into visual production.

Part 2

Content: Images and Video with AI

Two image models were tested to explore visual directions. Flux leaned toward a graphic, fashion-illustration aesthetic (Fig. 3), while Nano Banana produced more realistic, cinematic results (Fig. 4) — a better fit for this project.

Flux2 Pro — graphic and design-forward · Nano Banana Pro/2 — cinematic and true-to-life

Nanona was chosen to generate all images. Each image went through at least three rounds of iteration — a time-consuming but worthwhile process.

Step 1

Finding Scenes and Story Direction

The process started with a 3×3 grid — generating 9 images at once to quickly explore scene possibilities and save time. At this stage, the main criteria were relevance to the theme and whether each image felt like it had a story behind it.

The following prompt was used to place the character across a variety of settings:

“Create a full Instagram-style grid composed of nine different portrait images. Each of the 9 images should present a unique concept, outfit, pose, and environment that fits a stylish, modern aesthetic.

Include a mix of:

  • Cinematic portraits
  • Fashion/streetwear scenes
  • Close-up beauty shots
  • Minimalist studio portraits”
Generating a 3×3 grid of 9 images in one go using Nano Banana Pro/2 on Freepik
Step 2

Multiple Shots Within the Same Scene

After generating a large pool of scenes, the best images were selected and expanded into a 2×3 grid of 6 shots. The focus here shifted from scene variety to variations within the same scene — different angles, emotional beats, and story moments. Note: the phrase "20-second cinematic clip" in the prompt below isn't asking for a video — it's instructing Nano Banana to think in terms of cinematic narrative logic across a series of still images.

Prompt: "Expand the image into a 20-second cinematic clip with a clear theme and emotional progression (setup → build → turn → payoff)."

Generating a 2×3 grid of 6 shots using Nano Banana Pro/2 on Freepik
Step 3

Upscaling and Refining

Promising images from the previous step were upscaled, then refined across multiple variations. The strongest version of each was selected for the final site.

Selected images go through continuous rounds of refinement

With all 16 images finalized, attention turned to the hero video — the background for the large banner at the top of the page, the first thing visitors see. It shows the woman receiving an invitation letter, echoing the concept of being drawn into each event.

Generating the hero video using Seedance2 on the Dreamina platform

With copy, images, and video all in place, it was time to start building with Claude Code.

Part 3

Code: Building and Publishing with Claude Code

The original plan was to use Google Stitch for visual design and Claude Code for the code. Stitch turned out to be too limiting for detailed work, so it was dropped. Claude Code handled everything instead, guided by web-specific Skills — predefined visual specification documents that give Claude a design system to follow.

Claude Code was installed locally and tested with a simple CLI prompt first. Once confirmed working, the build moved into VS Code with the Claude Code extension and Skills loaded as reference. VS Code was chosen over Antigravity — a very similar VS Code fork — simply for familiarity.

Having a Skill doesn't guarantee a good-looking result. Aesthetic judgment and UI/UX experience still matter. AI lowers the barrier to entry, but quality depends on the person making the calls.

Claude Code running in VS Code. The diff view shows before (left) and after (right) — making every change easy to review. Designers without a coding background may not recognize CSS or JS, but that's perfectly fine. Claude Code handles all of it automatically.

Around the 80% mark, prompting for small adjustments started feeling slow. At that point, editing CSS and JS directly was faster than describing each change to Claude Code.

The biggest advantage AI brings to web production is speed of problem-solving. What once meant long detours through Google and Stack Overflow can now be resolved in a conversation — leaving more time for experimentation and refinement. The finished page was published to GitHub Pages.

That covers the full process — from concept to a live, published page, built with AI.

Despite heavy AI involvement, human judgment remained essential throughout. The theme, visual direction, and interaction design all required deliberate creative decisions. As agentic workflows mature, full pipeline automation may become possible — but AI tends toward the aesthetic average. For a project like this, human intervention isn't optional. It's what makes it distinctive.
